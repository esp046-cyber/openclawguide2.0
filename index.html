Quickstart

Get a working OpenClaw agent running in under 5 minutes. Every code block is ready to copy.

STEP 1 Install OpenClaw

Requires Python 3.11+ and pip. One command installs the core framework plus the CLI.

pip install openclaw

Verify the install:

openclaw --version # openclaw 2.2.0 (claude-sonnet-4-6 default)

Need a specific version or extras?

pip install "openclaw[rag,events]==2.2.0" # [rag] → Qdrant + sentence-transformers # [events] → Redis Streams + Kafka adapter # [all] → everything
STEP 2 Set your Anthropic API key

OpenClaw uses Claude as the default model. Export your key — get one free at console.anthropic.com.

# macOS / Linux export ANTHROPIC_API_KEY="sk-ant-..." # Windows PowerShell $env:ANTHROPIC_API_KEY="sk-ant-..." # .env file (recommended for projects) ANTHROPIC_API_KEY=sk-ant-...

Or pass it inline:

from openclaw import Agent agent = Agent(api_key="sk-ant-...")
STEP 3 Your first agent — hello_agent.py

Copy this entire file. It creates a basic research agent, runs it, and prints the result.

from openclaw import Agent, tools import asyncio # 1. Define your agent agent = Agent( name="hello_agent", model="claude-sonnet-4-6", # or claude-haiku-4-5 for speed system="You are a helpful research assistant.", tools=[tools.web_search], # add more tools as needed max_steps=5, ) # 2. Run it async def main(): result = await agent.run( "What is the latest version of Python and when was it released?" ) print(result.output) print(f"Steps taken: {result.steps}") print(f"Tokens used: {result.usage.total_tokens}") asyncio.run(main())
# Run it python hello_agent.py
STEP 4 Add memory (remember across turns)

Without memory, each agent.run() call starts fresh. Add memory="session" to carry context across turns.

agent = Agent( name="chat_agent", model="claude-sonnet-4-6", memory="session", # keeps last N turns in context memory_limit=20, # max turns before auto-summarization ) # Turn 1 r1 = await agent.run("My name is Alex.") # Turn 2 — agent remembers "Alex" r2 = await agent.run("What did I just tell you?") print(r2.output) # → "You told me your name is Alex."
memory="none" → stateless memory="session" → this run only memory="persistent" → survives restarts
STEP 5 Scaffold a full project

The CLI generates a batteries-included project with config, agents, and a test suite.

openclaw init my-project cd my-project
# Generated structure: my-project/ ├── openclaw.yaml # global config ├── agents/ │ ├── classifier.py # intent router │ ├── research.py # web + rag agent │ └── synthesis.py # result aggregator ├── workflows/ │ └── main.yaml # orchestration DAG ├── tools/ │ └── custom_tool.py # add your own tools here ├── tests/ │ └── test_agents.py └── .env # ANTHROPIC_API_KEY goes here
# Start the dev server (hot reload) openclaw dev

AVOID Top 5 beginner mistakes
1 · No max_steps limit
Without max_steps agents can loop forever burning tokens. Default is 20. Set it explicitly for each agent type.
Agent(max_steps=5) # fast classifiers Agent(max_steps=15) # research agents Agent(max_steps=30) # long-running pipelines
2 · Forgetting await
All agent calls are async. Forgetting await returns a coroutine object, not the result.
# ✗ Wrong — result is a coroutine, not output result = agent.run("hello") # ✓ Correct result = await agent.run("hello")
3 · Putting secrets in code
Never hardcode API keys. Use environment variables or a .env file with python-dotenv.
pip install python-dotenv
from dotenv import load_dotenv load_dotenv() # reads .env automatically
4 · Using haiku for complex reasoning
claude-haiku-4-5 is fast and cheap but struggles with multi-step reasoning. Use claude-sonnet-4-6 for research and orchestration.
5 · No error handling on tool calls
External tools fail. Wrap them in try/except and return structured error dicts so the agent can reason about failures instead of crashing.
async def my_tool(query: str) -> dict: try: data = await fetch_api(query) return {"ok": True, "data": data} except Exception as e: return {"ok": False, "error": str(e)}

Model Routing

OpenClaw's model router dynamically selects between locally-hosted models (Ollama, llama.cpp, vLLM) and cloud API endpoints (OpenAI, Anthropic, Cohere) based on task complexity, latency requirements, and cost thresholds.

Local models provide sub-10ms TTFT for simple completions while preserving data sovereignty. API models handle complex reasoning, large context windows, and specialized capabilities.

# Model router config model_router: strategy: hybrid local_threshold_tokens: 4096 api_fallback: gpt-4o cost_cap_usd: 0.05

Service routes are defined declaratively. Each route specifies a trigger pattern, target agent pool, and optional middleware pipeline (auth, rate limiting, context injection).

Routes support regex matching, header inspection, and payload-based branching enabling fine-grained control over request dispatch.

The Agent Runtime manages task queuing, tool invocation, and result aggregation. Each agent instance is isolated with its own memory context and tool bindings.

Tasks support parallel fan-out, sequential chaining, and conditional branching. Results are automatically merged using configurable aggregation strategies.

agent: id: research_agent_01 tools: [web_search, rag_retrieval, code_exec] memory: session max_steps: 12

CRITICALInfinite Loop Detection

          
Agent calls tool → tool emits event → event triggers same agent. Without cycle detection the swarm deadlocks consuming tokens until context limit.

          

            
class CycleGuard:
  def __init__(self, max_visits=3):
    self.visited = {}          # node_id → visit_count
    self.max_visits = max_visits

  def check(self, node_id: str) -> bool:
    self.visited[node_id] = self.visited.get(node_id, 0) + 1
    if self.visited[node_id] > self.max_visits:
      raise CycleError(f"Cycle detected: {node_id} visited {self.visited[node_id]}x")
    return True

# Inject into every agent dispatch
guard = CycleGuard(max_visits=2)
async def dispatch(agent_id, payload):
  guard.check(agent_id)       # raises on cycle
  return await agents[agent_id].run(payload)

CRITICALContext Window Overflow

          
Long-running agents accumulate tool outputs that silently exceed model context. The model starts hallucinating or truncating critical early instructions.

          

            

              
⚠ WITHOUT GUARD

              
Messages grow unbounded → 200k token context → model skips system prompt → jailbreak surface increases

            

            

              
✓ WITH GUARD

              
Token counter on each append → sliding window summarization at 70% capacity → preserve system + last N turns

            

          

          

            
CONTEXT_BUDGET = 180_000   # leave 20k for response
COMPRESS_AT   = 0.70       # trigger at 70% full

async def safe_append(messages, new_msg, tokenizer):
  tokens = tokenizer.count(messages + [new_msg])
  if tokens > CONTEXT_BUDGET * COMPRESS_AT:
    middle = messages[1:-4]    # keep system + last 4
    summary = await summarize(middle)
    messages = [messages[0], {"role":"assistant","content":summary}] + messages[-4:]
  messages.append(new_msg)
  return messages

HIGHRace Conditions — Parallel Tool Calls

          
Multiple agents write to the same memory key simultaneously; last-write-wins causes silent data loss. Also: two agents claim the same external resource lock.

          

            
# Optimistic locking via version vectors
async def atomic_write(key, value, expected_version):
  current = await memory.get(key)
  if current.version != expected_version:
    raise ConflictError(f"Version mismatch on {key}")
  await memory.set(key, value, version=expected_version+1)

# Distributed lock with TTL (Redis-backed)
async with DistributedLock(key="resource:db_write", ttl_ms=5000):

def validate_tool_call(call: dict) -> dict:
  name = call.get("name")
  if name not in tool_registry:
    return {"error": f"Unknown tool '{name}'. Available: {list(tool_registry)}"}
  schema = tool_registry[name].input_schema
  missing = [k for k in schema["required"] if k not in call["input"]]
  if missing:
    return {"error": f"Missing required fields: {missing}"}
  return call  # valid

PWAService Worker Edge Cases

          
Browser-specific failure modes that break offline functionality or the agent cache manager.

          

            

              
Blob SW scope limitation

              
Blob-URL service workers have blob:// as their origin. They cannot intercept https:// fetches. Fix: serve from localhost or HTTPS; the SW registers at the page origin, not blob.

            

            

              
Safari private browsing blocks CacheStorage

              
Safari in private mode throws SecurityError on caches.open(). Always wrap in try/catch; degrade to in-memory map.

            

            

              
Firefox rejects SyncManager entirely

              
Background Sync API is Chrome/Edge only. OpenClaw detects 'SyncManager' in window and disables the Register Sync button honestly rather than silently failing.

            

            

              
oc:// virtual URL cache collision

              
If two agents share an id string, their cache keys collide. Agent IDs are validated as globally unique slugs at registration time.

            

            

              
Storage quota eviction (iOS)

              
iOS evicts CacheStorage after 7 days of non-use in Safari. The SW re-caches all agent configs on every install event, so re-opening the app restores them automatically.

DIDAgent Decentralized Identifiers

          
Each OpenClaw agent gets a did:key or did:web at instantiation. This replaces shared API keys and enables per-agent audit trails, revocation, and cross-org trust.

          

            
from did_resolver import DIDKey

# Generate agent DID at instantiation
class AgentIdentity:
  def __init__(self, agent_id: str):
    self.keypair = Ed25519KeyPair.generate()
    self.did = DIDKey.from_public_key(self.keypair.public_key)
    # did:key:z6Mkf...  — deterministic from public key
    self.agent_id = agent_id

  def sign(self, payload: dict) -> str:
    # JWT signed with agent's private key
    return jwt.encode({
      "iss": self.did,
      "sub": self.agent_id,
      "iat": time(),
      "payload": payload
    }, self.keypair.private_key, algorithm="EdDSA")

  def verify_peer(self, token: str, peer_did: str) -> bool:
    public_key = DIDKey.resolve(peer_did).public_key
    return jwt.decode(token, public_key, algorithms=["EdDSA"])
